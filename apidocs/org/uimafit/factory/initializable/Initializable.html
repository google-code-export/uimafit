org.uimafit.factory.initializable
Interface Initializable

All Known Implementing Classes:
IntegerFileNamer
public interface Initializable

This interface provides a way of initializing a class with a UimaContext. The standard use case of this interface involves situations in which a member variable is declared with an interface type and the actual class that is used is decided at runtime. When the member variable is instantiated, it is useful to provide it a UimaContext so that it can initialize itself based on configuration parameters.

As an example, consider the component XWriter which has a member variable of type XWriterFileNamer which is an interface that provides a way for XWriter to come up with a name each file that is generated given a JCas. The default XWriterFileNamer that is provided, IntegerFileNamer, provides a single configuration parameter for specifying a prefix to each file name that is generated. This parameter is initialized in the IntegerFileNamer.initialize(org.apache.uima.UimaContext) method. This initialize method is called because XWriter instantiates its XWriterFileNamer member variable using a InitializableFactory.create(org.apache.uima.UimaContext, java.lang.String, java.lang.Class) method. Therefore, when XWriter is instantiated it should be given a value for the configuration parameter IntegerFileNamer.PARAM_PREFIX. See the unit tests for XWriter for a complete code example. Note that the implementation of XWriterFileNamer does not have to implement Initializable if it has no need for the initialize method.
